# Phase 1/2 评分体系重构设计

## 背景

当前 daily_workflow 的 Phase 1 混合了技术面和基本面评分，导致一个关键缺陷：**品种从底部开始反转（如涨停）后，技术面评分惩罚了价格回升，导致基本面逻辑完好的品种被错误过滤**。

典型案例：多晶硅(PS0) 4-10 在 watchlist（P1=+51, 区间位5%），4-13 涨停后完全从候选中消失。原因链：

1. Phase 1 技术面评分（价格位置、RSI）因价格回升而大幅失分 → P1 总分跌破阈值 25
2. Wyckoff Spring 检测要求缩量（rv < 1.5），拒绝了底部放量涨停 → 信号被标为"疑似"
3. SOS 检测要求有效 Spring 作为前序事件，疑似 Spring 不算 → SOS 也变为疑似

## 设计目标

1. **Phase 1 改为纯基本面筛选**：确定"关注哪些品种、什么方向"，不受价格波动影响
2. **Phase 2 承接全部技术面分析**：确定"时机和信号强度"，增强反转信号检测
3. **修复重复维度**：当前 RSI、Wyckoff、OI 在两个 Phase 中重复计算，需合并去重
4. **修复反转信号级联失败**：Spring 和 SOS 检测逻辑过严，错过底部放量反转
5. **保留 RRF 排名融合**：Phase 1（基本面排名）+ Phase 2（技术面排名）

## Phase 1 重构：纯基本面筛选

### 移除的维度（移至 Phase 2）

| 维度 | 原范围 | 移除原因 |
|------|--------|---------|
| 价格区间位 | ±30 | 技术指标，且"逆势"逻辑在价格反转时反向惩罚 |
| RSI(14) | ±15 | 技术指标，与 Phase 2 完全重复 |
| 均线趋势 | ±10 | 技术指标 |
| Wyckoff 量价综合 | ±25 | 技术指标，与 Phase 2 Wyckoff 分项重复 |

### 移除的维度（移至 Phase 2）

| 维度 | 原范围 | 移除原因 |
|------|--------|---------|
| 持仓结构(OI) | ±10 | 市场微观结构指标，更接近技术面 |

### 保留的基本面维度

| 维度 | 范围 | 数据源 | 说明 |
|------|------|--------|------|
| 库存变化 | ±20 | `get_inventory` (东方财富) | 4周变化 + 累库/去库周数 |
| 库存分位 | ±10 | `get_inventory` | 52周高低范围内的位置 |
| 仓单变化 | ±15 | `get_warehouse_receipt` (交易所) | 仓单增减比例 |
| 季节性 | ±10 | 日线历史数据 | 当前月份的历史涨跌概率 |
| 生猪专项 | ±15 | `get_hog_fundamentals` | 仅LH0：利润率 + 价格趋势 |

基本面总分范围：约 ±70（实际因数据覆盖不全，多数品种 ±30~50）。

### 阈值调整

原阈值 25 基于 tech(±80) + fund(±80) = ±160 的总分范围。纯基本面后总分范围缩至 ±70，阈值需相应降低。

新阈值策略：**`abs(fund_score) >= 10`**。理由：
- 至少有一个基本面维度给出明确信号
- 仅季节性(±10)的品种（如多晶硅无库存无仓单）也能刚好通过
- 若 `fund_score` 接近 0 说明基本面没有方向性信号，不值得深度分析

### 方向判定

`direction` 仍由 `fund_score` 的符号决定：正分 = 做多，负分 = 做空。

### OI 四象限 bug 修复

当前代码 `增仓上涨` 和 `增仓下跌` 都给 `fund_score += 5`，这不正确。

此维度移至 Phase 2 后自然解决（Phase 2 已有独立的 `analyze_oi` 实现，正确区分了四象限方向）。

### 数据覆盖不足的品种处理

10 个品种无库存数据映射（PS0, FB0, AO0, BC0, CY0, LU0, NR0, PX0, SC0, SH0），仓单覆盖也仅 20 个品种。

处理策略：**不做特殊处理**。如果一个品种没有可用的基本面数据，它的基本面分数自然趋近于 0，不会通过阈值 — 这是合理的：没有基本面数据支撑时不应该从基本面角度推荐它。但这些品种仍可通过 `--skip-screen` 模式手动指定分析。

### 价格位置维度设计（新增）

从 Phase 1 的"逆势打分"(低位=+30)改为"上下文连续值"：

```python
# 价格在近300日高低点间的位置（0~100）
# 不再做逆势判断，而是提供一个连续的价格上下文
# 极端位置（<10% 或 >90%）给予小幅加分，但方向与趋势一致而非对抗
if range_pct < 10:
    price_pos_score = 5   # 极低位，轻度看多
elif range_pct > 90:
    price_pos_score = -5  # 极高位，轻度看空
else:
    price_pos_score = 0   # 中间位置不贡献
```

权重仅 ±5，作为辅助参考而非主导因子。

## Wyckoff 反转信号改进

### Spring 检测：新增"带力弹簧"路径

当前 Spring 验证要求 `vol_ok = rv < 1.5`（缩量试探），拒绝所有放量 Spring。

在 Wyckoff 理论中，"Spring with power" / "shakeout" 是一种合法的变体：高量击破支撑后强力收回，反映主力在底部大量吸筹。

新增 priority=2 的"带力弹簧"路径：

```
条件：
  - 形态: low < prev_low AND close > prev_low（与经典 Spring 相同）
  - depth > 0.2%（与经典 Spring 相同）
  - 上下文: has_trading_range OR trend_slowed（与经典 Spring 相同）
  - 成交量: rv >= 1.5（放量，与经典 Spring 相反）
  - 收盘位: cp > 0.6（强力收回，比经典 Spring 的 0.5 更严格）

priority = 2（有效信号，可作为入场依据）
```

对称地，为 Upthrust 也新增"带力上冲回落"路径。

### SOS/SOW 检测：允许疑似前序事件

当前 SOS 要求 `seen_signals` 中有 SC/Spring/StopVol_Bull，但 `seen_signals` 只添加 priority >= 1 的事件。

改进：引入**弱前序匹配**：
- 如果有 priority >= 2 的前序事件 → SOS priority = 4（不变，强确认）
- 如果有 priority = 0 的疑似前序事件 → SOS priority = 2（新增，弱确认，降低置信度）
- 如果完全无前序事件 → SOS priority = 0（不变，疑似）

### 反转信号强度分级

当前 `assess_reversal_status` 返回 `has_signal: bool`（二元）。改为返回 `signal_strength: float`（0~1 连续值）：

| 情况 | signal_strength | 说明 |
|------|----------------|------|
| 无任何相关事件 | 0.0 | 完全无信号 |
| 仅有疑似事件(priority=0) | 0.1~0.2 | 有苗头但未确认 |
| 有预备信号(SC/BC/StopVol, priority>=1) | 0.3~0.4 | 已见底/顶迹象 |
| 有入场信号但不新鲜(>3天) | 0.4~0.5 | 信号老化 |
| 有新鲜带力弹簧(priority=2) | 0.6~0.7 | 可考虑入场 |
| 有新鲜经典Spring/UT(priority=3) | 0.7~0.8 | 经典入场点 |
| 有新鲜SOS/SOW(priority=4) | 0.8~0.9 | 强确认信号 |
| SOS/SOW + 有前序事件 | 0.9~0.95 | 最强信号组合 |

`has_signal` 保留为兼容字段：`signal_strength >= 0.5` 时为 True。

`actionable` 判定条件更新：
- 原：`has_signal AND |score| > 20 AND rr >= 1.0`
- 新：`signal_strength >= 0.5 AND |score| > 20 AND rr >= 1.0`

## RRF 融合保留

保留当前已实现的 RRF (Reciprocal Rank Fusion) 排名融合：

```
RRF(d) = 1/(k + rank_P1) + 1/(k + rank_P2)    k=10
```

- Phase 1 排名：按 `abs(fund_score)` 降序
- Phase 2 排名：按 `abs(tech_score)` 降序

排名逻辑不变，仅因 Phase 1 评分内容变化而自然调整。

## 多晶硅案例验证

重构后多晶硅(PS0)的预期行为：

**4-10（涨停前）**：
- Phase 1：fund_score ≈ +20（季节性 + OI 已移出） → 实际只有季节性偏空(-10)，需确认 → 如果季节性为唯一信号且 <10，可能不过阈值
- **修正**：PS0 无库存/仓单数据，季节性偏空。纯基本面后 PS0 可能无法通过 Phase 1

**这暴露了一个设计权衡**：纯基本面的 Phase 1 会丢掉像多晶硅这样"技术面极端但无基本面数据"的品种。

**解决方案**：在 Phase 1 中保留一个**极端价格标记**（不参与评分，但当 `range_pct < 5% 或 > 95%` 时自动将品种加入候选池，即使基本面分数不达标）。这不是评分维度，而是一个安全网机制。

## 风险与限制

1. **Phase 1 阈值需要调优**：纯基本面后通过率会变化，可能需要运行几天观察
2. **10个无基本面数据品种**：将依赖"极端价格标记"安全网进入分析流程
3. **带力弹簧可能增加误报**：高量假突破后收回也可能是正常波动而非反转
4. **signal_strength 的阈值**：0.5 作为 actionable 分界点需要实战验证
