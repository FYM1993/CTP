# 品种分类阈值与生猪数据容错设计文档

> **状态**: 待评审
> **日期**: 2026-04-15
> **目标**: 解决不同品种波动特性不一致导致的筛选迟钝问题，并增加生猪数据的鲁棒性。

---

## 1. 需求背景
目前所有期货品种在 Phase 1 筛选时共用统一的基本面分值门槛（|Score| ≥ 10）。
- **问题 1**: 生猪（LH0）等高敏感品种，微小的基本面变化（如利润率变动 1%）即具有重要意义，10 分的门槛过高。
- **问题 2**: 生猪数据依赖外部 API，若养殖成本数据抓取失败，导致基本面分数归零，即便价格处于历史极低位也会落榜。
- **问题 3**: 报告文案中“方向”来源不透明，导致用户对“分数方向”与“交易方向”的冲突感到困惑。

## 2. 解决方案

### 2.1 品种分类与配置化阈值
在 `config.yaml` 中引入分类门槛，取代单一的硬编码门槛。

**分类标准**:
- **高敏感度 (High Sensitivity)**: 微变即信号。例：生猪。
- **农产品 (Agricultural)**: 受天气影响大，波动中等。例：豆粕、玉米。
- **低敏感度 (Low Sensitivity)**: 金融属性强或新品种，波动剧烈。例：贵金属、碳酸锂。
- **默认 (Default)**: 黑色系、化工等。

### 2.2 生猪数据容错机制 (选项 B)
在生猪专项数据缺失时，利用 K 线数据（区间位 `range_pct`）进行动态补偿。

**补偿规则**:
- `range_pct < 5%` (历史极值): 补偿 **+10分**。
- `range_pct < 15%` (低位): 补偿 **+5分**。
这确保了只要生猪处于历史价格低位，即使 API 故障，也能进入 Phase 2 的反转信号监控。

### 2.3 入池理由与文案统一
- **入池理由 (entry_pool_reason)**: 在候选池中记录具体入选原因（如：`基本面达标(≥3，当前+5)` 或 `极端价位(区间位3%)`）。
- **文案调整**: 将报告中的“等待信号”改为“系统提示”，并明确标注“逆势”标签。

## 3. 技术实现方案

### 3.1 配置层 (`config.yaml`)
```yaml
fundamental_screening:
  default_threshold: 10
  categories:
    high_sensitivity:
      threshold: 3
      symbols: ["LH0"]
    agricultural:
      threshold: 8
      symbols: ["A0", "M0", "RM0", "C0", "CS0", "CF0", "SR0"]
    low_sensitivity:
      threshold: 15
      symbols: ["AU0", "AG0", "LC0", "SI0"]
```

### 3.2 逻辑层 (`scripts/daily_workflow.py`)
1. **`phase_1_screen`**: 
   - 加载配置中的类别映射。
   - 对每个品种，先查类别阈值，查不到则用 `default_threshold`。
2. **`_score_symbol`**:
   - 增加生猪补偿逻辑：若 `hog` 字典中 `profit_margin` 缺失且 `sym == 'LH0'`，执行 `range_pct` 补偿。
3. **入池理由记录**: 
   - 在 `result` 字典中增加 `entry_pool_reason` 字段。

### 3.3 数据层 (`scripts/data_cache.py`)
- 优化 `get_hog_fundamentals`: 确保局部 API 失败时，返回已拿到的部分数据并标记 `data_status: "partial"`。
