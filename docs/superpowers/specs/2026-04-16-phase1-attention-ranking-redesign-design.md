# Phase 1 关注优先级重构设计

> 状态：待用户复核
> 日期：2026-04-16
> 主题：将 Phase 1 从“基本面定方向”重构为“基本面发现机会”，并统一市场数据源口径

## 1. 背景

当前 Phase 1 的职责混杂了三件事：

- 基本面筛选
- 交易方向判断
- 候选池排序

这导致两个长期问题：

1. `Phase 1` 会把“未来可能反转的大机会”和“当前已经走强的顺势机会”压到同一条正负分轴上，语义互相冲突。
2. 用户实际关心的是“哪些品种值得进入 Phase 2 深挖”，但系统当前输出的是“一个带方向的分数”，这会导致底部机会、顶部机会、趋势机会互相挤压。

从近期现象看，生猪、多晶硅这类“已经显著偏离长期均衡、但短期技术面尚未完全确认”的品种，容易出现：

- 技术上进入了候选池，但用户感知上“像是没被筛出来”
- 极端低位保护只是准入条件，不是排序加分，导致重要机会被埋在候选池中后部
- `Phase 1` 的方向判断和用户真正希望的“机会发现”目标不一致

因此，本次重构的目标不是继续优化旧分数，而是重新定义 `Phase 1` 的职责边界。

## 2. 设计目标

新的 `Phase 1` 只回答两个问题：

1. 这个品种现在是否值得进入 `Phase 2`
2. 它值得进入 `Phase 2` 的原因更偏向“反转机会”还是“趋势机会”

明确不再由 `Phase 1` 回答：

- 现在应不应该做多
- 现在应不应该做空
- 最终交易方向是什么

本次设计需要同时满足以下目标：

- 尽量别漏掉未来可能反转的大机会
- 尽量别漏掉当前已经走强、仍可能延续的强势品种
- 统一一个总池子，按关注优先级排序
- 总池子支持 `Top N` 配置化截断
- 保留清晰的标签和解释，不再让用户猜测分数语义
- 全项目的价格、K线、持仓量数据统一口径，优先使用 `TqSdk`

## 3. 术语统一

本设计统一使用中文术语，避免继续在业务规则中混用英文命名。

### 3.1 分数名称

- 反转机会分：衡量该品种偏离长期均衡后，未来可能发生再平衡或均值回归的机会强度
- 趋势机会分：衡量该品种当前供需方向是否已经进入顺风阶段，适合继续跟踪的机会强度
- 关注优先级分：衡量该品种当前有多值得进入 `Phase 2` 深挖
- 数据覆盖度：衡量当前分数基于多少有效因子得到，覆盖度越低，不确定性越高

### 3.2 标签名称

- 反转候选
- 趋势候选
- 双标签候选
- 数据覆盖不足

### 3.3 状态标签

状态标签不是交易建议，只用于解释基本面所处状态：

- 低位出清
- 高位扩产
- 紧平衡强化
- 过剩深化

## 4. 新的 Phase 1 职责定义

新的 `Phase 1` 定位为“基本面机会发现层”。

输入：

- 行情与K线数据
- 持仓量数据
- 基本面扩展数据

输出：

- 总池子候选列表
- 每个品种的反转机会分
- 每个品种的趋势机会分
- 每个品种的关注优先级分
- 标签
- 状态标签
- 数据覆盖度
- 中文摘要说明

不再输出：

- `direction`
- “做多候选 / 做空候选”这类交易导向语言

## 5. 数据源策略

### 5.1 总原则

项目内所有“价格 / K线 / 持仓量 / 结算 / 排名”等市场数据，统一以 `TqSdk` 为主口径。

项目内所有“库存 / 仓单 / 成本 / 利润 / 产业专项”等基本面补充数据，优先使用 `TqSdk EDB`；如果 `EDB` 不覆盖或权限不可用，再退回 `AkShare`。

即：

1. `TqSdk`：主行情真相源
2. `TqSdk EDB`：非价量基本面优先源
3. `AkShare`：基本面兜底源

### 5.2 使用范围

`TqSdk` 负责：

- 主连或连续合约价格
- 日线K线
- 分钟线K线
- Tick
- 持仓量
- 合约静态信息
- 排名数据
- 结算相关数据

`TqSdk EDB` / `AkShare` 负责：

- 库存
- 仓单
- 成本
- 利润
- 现货价格
- 生猪专项等产业数据
- 宏观或行业辅助指标

### 5.3 设计约束

- 不允许在业务层同时直接调用 `TqSdk` 和 `AkShare` 的行情接口
- 不允许让 `Phase 1` 评分逻辑直接依赖不同口径的价格数据
- 价格序列、历史区间、成本偏离等判断必须建立在统一行情源之上
- `TqSdk` 合约映射应单独封装，不能把字符串拼接规则散落在业务逻辑中

## 6. 评分体系总览

新的 `Phase 1` 不再用单一正负分轴，而是拆成两类机会分：

- 反转机会分
- 趋势机会分

最终统一生成：

- 关注优先级分

三个分数的关系如下：

- 反转机会分：看“偏离长期均衡后，有没有反转机会”
- 趋势机会分：看“供需方向是否已经进入顺风阶段”
- 关注优先级分：看“这个品种是否值得进入 Phase 2 深挖”

## 7. 反转机会分设计

反转机会分衡量：该品种偏离长期均衡后，未来是否存在再平衡和均值回归机会。

内部拆为两个方向性的子判断，但对外不输出方向建议：

- 向上修复机会：低于长期均衡后，未来向上修复的机会强度
- 向下回归机会：高于长期均衡后，未来向下回归的机会强度

最终反转机会分取两者中的较强者。

### 7.1 因子类别

#### 1. 长期均衡偏离因子

用于回答：当前价格、利润、库存是否已经偏离可持续的正常区间。

候选因子：

- 价格相对成本的偏离
- 价格在近300日区间中的位置
- 价格在全样本区间中的位置
- 利润率在历史分布中的位置

#### 2. 失衡持续性因子

用于回答：这种极端状态是短期波动，还是已经持续足够久。

候选因子：

- 连续亏损持续时间
- 连续高利润持续时间
- 持续低利润或高利润状态的长度

#### 3. 出清 / 扩产压力因子

用于回答：当前失衡是否已经开始逼迫行业收缩，或者刺激行业扩张。

候选因子：

- 深亏状态下的补栏或开工收缩压力
- 暴利状态下的扩产冲动
- 产业链供给端是否存在被动出清或主动扩产迹象

#### 4. 周期极端背景因子

用于回答：当前是否处在典型周期的极端区域。

候选因子：

- 历史库存极端
- 历史利润极端
- 价格长期极端位置
- 品种专项周期状态

### 7.2 核心原则

- “价格低”本身不直接等于高质量反转机会
- 只有“价格低 + 接近成本线 / 深亏 / 持续失衡 / 出清压力增强”组合出现时，才应形成强反转机会分
- 同理，“价格高”也只有在“暴利 / 扩产压力 / 长期高位不可持续”时，才构成高位反转机会

## 8. 趋势机会分设计

趋势机会分衡量：当前供需方向是否已经进入顺风阶段，值得继续跟踪。

内部同样拆为两个方向性的子判断，但不对外输出交易方向建议：

- 上行趋势机会：供需向紧平衡强化
- 下行趋势机会：供需向过剩深化

最终趋势机会分取两者中的较强者。

### 8.1 因子类别

#### 1. 库存流向因子

候选因子：

- 4周库存变化
- 8周累库 / 去库持续性

#### 2. 仓单流向因子

候选因子：

- 仓单增减比例
- 仓单方向延续性

#### 3. 利润与景气因子

候选因子：

- 利润改善或恶化方向
- 现货与利润是否同向验证供需变化

#### 4. 季节性与体制辅助因子

候选因子：

- 季节性倾向
- 品种专项体制状态

### 8.2 核心原则

- 趋势机会分描述的是“供需方向是否已经进入顺风阶段”
- 趋势机会分不依赖技术指标，不依赖 K 线形态，不依赖 OI 交易结构判断
- 所有技术确认都属于 `Phase 2`

## 9. 数据覆盖度与标准化

不同品种的数据覆盖并不相同：

- 有些品种有库存、仓单、利润、专项数据
- 有些品种只有部分库存或季节性
- 有些品种只有少量可用基本面指标

因此，本设计要求：

1. 所有因子统一标准化到 `0-100`
2. 缺失因子不按 `0` 处理
3. 只在当前可用因子上重分配权重
4. 输出数据覆盖度，表示当前分数基于多少有效信息得到
5. 对低覆盖度分数做轻度收缩，避免单一因子异常把品种抬得过高

标准化后的语义建议统一为：

- `0-30`：弱
- `30-60`：中
- `60-80`：强
- `80-100`：极强

## 10. 标签规则

标签由反转机会分和趋势机会分共同决定：

- 反转机会分达到阈值，趋势机会分未达到阈值：标记为“反转候选”
- 趋势机会分达到阈值，反转机会分未达到阈值：标记为“趋势候选”
- 两者都达到阈值：标记为“双标签候选”

状态标签由子分支决定：

- 低位出清
- 高位扩产
- 紧平衡强化
- 过剩深化

状态标签用于解释基本面所处阶段，不用于给出做多 / 做空建议。

## 11. 关注优先级分设计

关注优先级分用于统一总池子排序。

### 11.1 设计原则

- 两类机会必须都能公平进入总排序
- 同时具备反转与趋势特征的品种，应得到融合优势
- 但单边极强的品种，也不能因为另一边较弱而被埋掉

### 11.2 计算规则

关注优先级分由两部分组成：

#### 1. 倒数排名融合分

分别对以下两项做全市场排名：

- 反转机会分
- 趋势机会分

然后对两个排名做倒数排名融合，得到融合分。

该融合分用于奖励“两类机会都不差”的品种。

#### 2. 单边强化加分

如果一个品种在某一类机会上的分数特别高，且明显强于另一类机会分，则额外给予单边强化加分。

该加分用于保护：

- 单边极强的反转机会
- 单边极强的趋势机会

这样可以避免“另一侧不强”导致真正的大机会被排到很后。

#### 3. 数据覆盖度收缩

最终关注优先级分应受到数据覆盖度的轻度收缩。

目的：

- 允许低覆盖品种进入候选池
- 但降低它们冲到前排时的误判风险

## 12. 候选池规则

总池子只保留一个，不再分“做多池 / 做空池”。

### 12.1 入池条件

如果一个品种的“反转机会分”或“趋势机会分”中，较高的一项达到基础阈值，则进入候选池。

直白含义：

- 不要求它同时具备两类机会
- 只要它在某一类机会中足够强，就不要轻易漏掉

### 12.2 特例放行

如果某个品种数据覆盖不完整，但某一类机会分极高，也允许进入候选池，同时打上“数据覆盖不足”标签。

直白含义：

- 数据不全不等于一定没有机会
- 但需要明确标识该判断证据不完整

### 12.3 排序与截断

对全部候选池品种统一按关注优先级分排序。

最终只保留配置中的前 `N` 个品种进入 `Phase 2`。

这里的 `N` 应配置化。

## 13. Phase 2 接口变化

由于新的 `Phase 1` 不再输出方向，`Phase 2` 必须改成自决方向，而不是继续接收 `Phase 1` 的 `direction`。

新的 `Phase 2` 输入应包含：

- 品种代码
- 品种名称
- 反转机会分
- 趋势机会分
- 关注优先级分
- 标签
- 状态标签
- 数据覆盖度
- 中文摘要

新的 `Phase 2` 负责：

- 判断当前更适合做多、做空，还是仅观察
- 判断当前是否存在足够新鲜的技术确认
- 输出交易方向、入场条件、止损止盈等结果

职责边界变为：

- `Phase 1`：发现值得深挖的品种
- `Phase 2`：决定交易方向和时机

## 14. 模块边界设计

为避免继续让 `daily_workflow.py` 膨胀，新的 `Phase 1` 应按以下边界拆分：

### 14.1 `phase1_models.py`

职责：

- 定义结构化数据模型

建议模型：

- 因子集合
- 分数拆解结构
- 候选结果结构

### 14.2 `market_data_tq.py`

职责：

- 只负责通过 `TqSdk` 获取价格、K线、持仓、主连信息等市场数据
- 输出统一字段格式

### 14.3 `fundamental_data.py`

职责：

- 优先通过 `TqSdk EDB` 获取非价量数据
- 如无覆盖再退回 `AkShare`
- 负责屏蔽底层数据源差异

### 14.4 `phase1_factors.py`

职责：

- 将价格数据和基本面数据转换为标准化因子
- 不负责排序
- 不负责总分融合

### 14.5 `phase1_scoring.py`

职责：

- 计算反转机会分
- 计算趋势机会分
- 计算关注优先级分
- 打标签

### 14.6 `phase1_pipeline.py`

职责：

- 跑全市场
- 调用数据层与评分层
- 排序
- 截取前 `N`
- 输出给 `Phase 2`

### 14.7 `daily_workflow.py`

职责：

- 只做编排
- 不再承载 `Phase 1` 复杂评分细节

## 15. 对现有问题的直接修复效果

### 15.1 生猪场景

旧逻辑的问题：

- 一旦价格从极端低位反弹一天，`300日区间位` 很快脱离极端区域
- “极端低位”只是准入条件，不参与优先排序

新逻辑的效果：

- 只要仍处在深亏、接近成本线、供给被迫出清的状态，反转机会分仍然可以维持高位
- 不再依赖“是否还在 300 日极端低位”这一单一条件

### 15.2 多晶硅场景

旧逻辑的问题：

- 基本面方向与底部机会混在一起，容易把“已经深度失衡但短期趋势未反转”的品种压后

新逻辑的效果：

- 即便当前仍偏过剩，只要长期均衡偏离足够明显，反转机会分也可维持较高
- 如果供需继续恶化，趋势机会分也能同时体现，形成双标签候选

## 16. 风险与限制

### 16.1 `TqSdk EDB` 覆盖范围需逐项确认

虽然本地 `TqSdk 3.9.3` 已存在 `query_edb_data()`，且官方文档明确支持非价量数据服务，但当前项目实际使用的以下字段仍需逐项确认：

- 库存
- 仓单
- 成本
- 利润
- 生猪专项数据

设计结论：

- 不假定 `EDB` 完全覆盖当前所有 `AkShare` 基本面字段
- 先做映射确认，再决定哪些字段完全迁移，哪些字段保留兜底

### 16.2 单边强化加分参数需回测或样本验证

如果单边强化加分过大，会导致只靠单类机会就冲到榜首；过小，又会重新埋掉真正的单边大机会。

因此参数应在实现阶段通过历史样本验证后再固化。

### 16.3 低覆盖品种需要显式暴露不确定性

不能因为数据不全就直接剔除所有品种，但必须让“数据覆盖不足”的风险清楚可见。

## 17. 实施前必须确认的事项

1. `TqSdk EDB` 与现有 `AkShare` 字段的映射清单
2. 各品种专项因子适配器的边界
3. 关注优先级分的参数与阈值
4. `Phase 2` 如何从“无方向输入”改为“自决方向”

## 18. 最终结论

本次重构的核心不是“调一调原来的 Phase 1 分数”，而是改变 `Phase 1` 的问题定义：

- 旧问题：基本面应不应该直接决定多空方向
- 新问题：哪些品种最值得进入 Phase 2 深挖

在这个新定义下：

- 反转机会和趋势机会可以同时保留
- 总池子排序可以统一
- `Top N` 可以公平截断
- 价格、K线、持仓量的口径可以统一到 `TqSdk`
- 非价量基本面可以优先尝试 `TqSdk EDB`，缺失部分再由 `AkShare` 兜底

这套设计更符合当前项目的真实目标，也更符合用户希望“尽量别漏掉未来大机会，同时别漏掉当前强势品种”的要求。
